# Product Spec / PRD 模板

## 1. 原始需求
- 用户的一句话需求
- 当前任务类型：新建规格 / 迭代更新

## 2. 产品概述
- 产品名称
- 一句话定位
- 主要用户
- 核心任务
- 一期范围概述

## 3. 需求澄清纪要
### 已确认决策
- 已确认项

### 待确认项
- `[待确认]` 项

### 被排除方案
- 明确不采用的方案

### 关键假设
- 当前暂按什么假设推进

## 4. 目标用户与使用场景
### 主要用户
- 谁使用
- 使用前提

### 核心场景
- 场景 1
- 场景 2
- 场景 3

## 5. 产品范围
### v1 必做范围
- 一期必须成立的内容

### 非目标
- 当前明确不做的内容

### 后续版本
- 延后处理的内容

## 6. 核心模型
- 主要实体
- 实体关系
- 关键状态
- 关键状态流转

## 7. 功能设计
对每个主要功能写清楚：
- 功能名称
- 目的
- 触发角色
- 输入
- 输出
- 核心规则
- 边界情况

## 8. 业务规则
- 校验规则
- 状态流转规则
- 搜索 / 筛选 / 排序规则
- 删除 / 归档 / 恢复规则
- 权限规则
- 异常处理规则

## 9. 用户流程
按步骤写出一期主路径，例如：
- 首次进入
- 创建 / 导入
- 浏览 / 搜索
- 编辑 / 更新
- 完成 / 归档 / 删除

补充必要的分支流程：
- 零状态流程
- 异常流程
- 回退流程

## 10. 信息架构
只有产品包含用户界面时才写这一节。

- 主要页面或交互面
- 导航结构
- 默认入口
- 桌面端 / 移动端差异

## 11. 页面级说明
只有产品包含用户界面时才写这一节。

对每个主要页面说明：
- 页面名称
- 页面目的
- 关键字段
- 关键动作
- 页面状态
- 与其他页面的跳转关系

## 12. 原型与界面说明
只有产品包含用户界面时才写这一节。

至少写清楚：
- 页面布局结构
- 主次操作层级
- 零状态 / 空状态 / 加载态 / 错误态
- 关键组件
- 线框说明文件或 AI 原型提示词链接

## 13. 数据与系统说明
- 必需字段
- 派生字段
- 外部集成项
- 运行或部署前提
- 如果是无界面产品，可在这里补充触发方式、上下游流程、外部输入输出

注意：
- 这一节只写会影响产品设计的系统约束
- 不写实现方案、接口设计或代码结构

## 14. AI 功能
只有产品真的包含 AI 能力时才写这一节。

每个 AI 功能至少写清楚：
- 用户价值
- 触发位置
- 输入
- 输出
- 约束
- 失败处理

## 15. 开放问题
- `[待确认]` 项

## 16. 验收标准
- 用具体可判断的语句定义完成条件
- 验收标准应与产品行为对应，而不是实现方式
