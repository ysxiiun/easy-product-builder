# UI Spec 模板

## 1. 文档范围
- 本文档对应的产品或功能
- 本次覆盖的页面范围
- 不在本次 UI 范围内的页面或交互

## 2. 视觉方向总览
- 用户明确偏好
- AI 推荐方向
- 当前假设
- 不采用的方向

注意：
- 如果用户尚未拍板，明确标注 `[待确认]`
- 不要把未追问的视觉内容写成既定事实

## 3. 主题色与色彩策略
- 品牌主色或主题色组合
- 功能色规则：成功、警告、错误、信息
- 高亮、禁用、边框、背景的用色原则
- `[待确认]` 的色彩项

## 4. 风格关键词与界面气质
- 风格关键词，例如专业理性、效率工具感、亲和轻量、数据驾驶舱、品牌感
- 视觉气质说明
- 参考产品或参考设计
- 明确不希望出现的风格

## 5. 布局原则与信息密度
- 终端类型：桌面端 / 移动端 / 响应式
- 默认入口
- 导航方式：顶部导航 / 侧边栏 / Tab / 单页流 / 其他
- 布局倾向：信息密集 / 卡片化 / 表单优先 / 列表优先 / 工作台式 / 沉浸式
- 页面密度：轻量 / 标准 / 信息密集
- 全局交互原则

## 6. 组件与模块表现倾向
- 更强调的模块类型：表格 / 面板 / 图表 / 卡片 / 表单 / 时间轴 / 其他
- 组件气质说明
- 是否有设计系统约束
- 品牌或实现层禁忌

## 7. 信息架构与页面清单
- 页面 1：页面目标
- 页面 2：页面目标
- 页面 3：页面目标
- 页面之间的关系

## 8. 页面级说明
对每个页面分别说明：

### 页面名称
- 页面目标
- 进入路径
- 适用角色
- 页面结构
- 主区域
- 辅助区域
- 默认展示内容
- 主操作
- 次操作
- 关键字段
- 与其他页面的跳转关系
- 本页对应的视觉重点

## 9. 页面状态
- 零状态
- 空结果状态
- 加载态
- 错误态
- 成功反馈
- 权限受限状态

## 10. 关键交互
- 页面跳转关系
- 弹窗 / 抽屉 / 二级页面触发条件
- 表单提交流程
- 列表与详情联动关系
- 批量操作与高风险确认机制
- 需要确认的高风险交互

## 11. 跨端与特殊约束
- 桌面端与移动端差异
- 权限差异
- 极端数据量下的展示约束
- 暂未定稿项

## 12. 原型提示词附录
只有用户明确要求给原型工具或设计 AI 的提示词时才写这一节。

- 页面范围
- 目标用户
- 导航结构
- 视觉方向
- 主题色策略
- 关键区域与组件
- 主次操作
- 必须覆盖的页面状态
- 不允许擅自扩展的内容
