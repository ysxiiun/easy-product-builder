# Product Spec 模板

## 1. 产品概述
- 产品名称
- 一句话定位
- 核心问题
- 主要用户

## 2. 目标与非目标
### 目标
- 一期必须达成什么

### 非目标
- 明确不做什么

## 3. 目标用户与场景
### 主要用户
- 谁使用
- 为什么使用

### 核心场景
- 场景 1
- 场景 2
- 场景 3

## 4. 核心模型
- 主要实体
- 实体关系
- 关键状态
- 关键状态流转

## 5. 功能范围
对每个主要功能写清楚：
- 目的
- 输入
- 输出
- 主要规则
- 边界情况

## 6. 业务规则
- 校验规则
- 状态流转规则
- 搜索 / 筛选 / 排序规则
- 删除 / 归档 / 恢复规则
- 权限规则

## 7. 用户流程
按步骤写出一期主路径，例如：
- 首次进入
- 创建 / 导入
- 浏览 / 搜索
- 编辑 / 更新
- 完成 / 归档 / 删除

## 8. 信息架构
- 主要页面或交互面
- 导航结构
- 默认入口
- 桌面端 / 移动端差异（如果有）

## 9. 页面级说明
对每个主要页面说明：
- 页面目的
- 关键字段
- 关键动作
- 页面状态

## 10. 数据与系统说明
- 必需字段
- 派生字段
- 集成项
- 部署前提

## 11. AI 功能
只有产品真的包含 AI 能力时才写这一节。

每个 AI 功能至少写清楚：
- 用户价值
- 触发位置
- 输入
- 输出
- 约束
- 失败处理

## 12. 发布范围
### v1
- 一期必须做的内容

### 后续版本
- 明确延后的内容

## 13. 开放问题
- `[待确认]` 项

## 14. 验收标准
- 用具体可判断的语句定义完成条件
