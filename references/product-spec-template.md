# Product Spec 模板

## 1. 原始需求
- 用户的一句话需求
- 当前任务类型：新建规格 / 迭代更新

## 2. 产品概述
- 产品名称
- 一句话定位
- 主要用户
- 核心任务
- 一期范围概述

## 3. 需求澄清纪要
### 已确认决策
- 已确认项

### 待确认项
- `[待确认]` 项

注意：
- 这里只记录已经被追问触达、但尚未定案的问题
- 不要把尚未问到的空白直接塞进 `[待确认]`

### 被排除方案
- 明确不采用的方案

### 关键假设
- 当前暂按什么假设推进

注意：
- 这里只记录用户明确不确定、拒答，或授权 AI 先代补的假设
- 不要把纯猜测写成关键假设

### 术语定义
- 关键业务术语及定义

## 4. 目标用户与使用场景
### 主要用户
- 谁使用
- 使用前提

### 核心场景
- 场景 1
- 场景 2
- 场景 3

## 5. 产品范围
### v1 必做范围
- 一期必须成立的内容

### 非目标
- 当前明确不做的内容

### 后续版本
- 延后处理的内容

## 6. 核心对象与状态
- 主要实体
- 实体关系
- 关键状态
- 状态流转规则

## 7. 功能设计
对每个主要功能写清楚：
- 功能名称
- 目的
- 触发角色
- 输入
- 输出
- 核心规则
- 边界情况

## 8. 业务规则
- 校验规则
- 状态流转规则
- 搜索 / 筛选 / 排序规则
- 删除 / 归档 / 恢复规则
- 权限规则
- 异常处理规则

## 9. 用户流程
按步骤写出一期主路径，例如：
- 首次进入
- 创建 / 导入
- 浏览 / 搜索
- 编辑 / 更新
- 完成 / 归档 / 删除

补充必要的分支流程：
- 零状态流程
- 异常流程
- 回退流程

## 10. 与 UI 的边界
只有产品包含用户界面时才写这一节。

- 哪些流程需要通过页面承载
- 需要哪些核心页面或交互面
- 哪些页面级细节会单独写入 `UI-Spec.md`

注意：
- 这里写产品层对界面的要求
- 不在这里展开页面布局、组件层级和逐页交互细节

## 11. 数据与系统约束
- 必需字段
- 派生字段
- 外部集成项
- 运行或部署前提
- 如果是无界面产品，可补充触发方式、上下游流程、外部输入输出

注意：
- 这一节只写会影响产品设计的系统约束
- 不写实现方案、接口设计或代码结构

## 12. AI 功能
只有产品真的包含 AI 能力时才写这一节。

每个 AI 功能至少写清楚：
- 用户价值
- 触发位置
- 输入
- 输出
- 约束
- 失败处理

## 13. 开放问题
- `[待确认]` 项

## 14. 验收标准
- 用具体可判断的语句定义完成条件
- 验收标准应与产品行为对应，而不是实现方式
