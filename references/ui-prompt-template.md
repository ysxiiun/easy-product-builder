# AI 原型提示词模板

用于将已经澄清过的界面需求，整理成可交给 AI 生成原型或高保真草图的提示词。

注意：
- 这不是默认输出文档
- 默认应先生成或更新 `UI-Spec.md`
- 只有当用户明确要求“给设计 AI / 原型工具的提示词”时，才基于 `UI-Spec.md` 生成这一部分内容
- 提示词必须同时继承 `UI-Spec.md` 中的页面结构、交互规则和视觉方向，不要只抄页面清单

## 提示词结构

### 1. 产品背景
- 产品名称
- 目标用户
- 核心使用场景
- 本次要生成的页面范围

### 2. 视觉方向
- 推荐的整体风格
- 主题色组合或品牌主色
- 信息密度与布局倾向
- 参考产品或参考设计
- 不允许出现的视觉方向

### 3. 页面与结构
- 页面名称
- 页面目标
- 导航结构
- 页面主要区域
- 各区域包含的组件

### 4. 关键信息与操作
- 页面默认展示的信息
- 最重要的主操作
- 次级操作
- 关键表单字段
- 列表字段或卡片字段

### 5. 状态要求
- 零状态
- 空结果状态
- 加载态
- 错误态
- 成功反馈
- 权限受限状态

### 6. 输出与约束
- 输出形式：低保真线框 / 中保真界面 / 高保真原型
- 页面数量
- 是否要求组件命名
- 是否要求标注交互说明
- 不要擅自扩展的内容

## 示例骨架

```text
请为一个{产品名称}生成{输出形式}。

目标用户是：{目标用户}。
核心场景是：{核心场景}。

整体视觉方向为：{视觉方向}。
主题色策略为：{主题色策略}。
布局倾向为：{布局倾向}。
参考风格可接近：{参考产品或参考设计}。
不要使用：{禁止出现的视觉方向}。

本次需要生成的页面包括：
1. {页面一}
2. {页面二}
3. {页面三}

整体导航结构为：{导航结构}。

其中 {页面一} 需要包含：
- {区域一}
- {区域二}
- {区域三}

页面默认展示的信息包括：
- {信息一}
- {信息二}

主操作为：{主操作}。
次操作为：{次操作}。

请补充以下状态：
- 零状态
- 空结果状态
- 加载态
- 错误态
- 权限受限状态

终端类型为：{终端类型}。
信息密度为：{信息密度}。

不要加入未定义的复杂业务模块，不要擅自扩展额外页面。
输出时请同时体现每个页面的结构、组件、交互目的和视觉风格约束。
```
